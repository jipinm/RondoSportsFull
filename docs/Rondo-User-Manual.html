Getting Started

Accessing the Admin Panel

  1. Open your web browser
  2. Navigate to: https://admin.yourdomain.com
  3. You will see the login page

Logging In

  1. Enter your credentials: Email and Password
  2. Click "Login"
  3. You will be redirected to the Dashboard
Change Password: You can change your password anytime. Choose a strong password (at least 8 characters, including uppercase, lowercase, numbers, and symbols).

Dashboard

The Dashboard provides an overview of your ticket booking system's performance at a glance. All metrics are sourced from the protected /admin/dashboard API endpoint and compare current performance with the previous equivalent period.

Today's Bookings

Confirmed bookings created today with a trend indicator against yesterday.

Weekly Bookings

Total confirmed bookings for the current week compared with the previous week.

Monthly Bookings

Bookings captured during the current month versus the prior month.

Total Revenue

Revenue from completed bookings with percentage change against last month.

Understanding Trend Indicators

  • Green upward arrow: Current period has grown compared to the previous period.
  • Red downward arrow: Current period has decreased compared to the previous period.
  • Gray dash: Insufficient data to calculate change (both periods are zero).

Charts and Activity

  • Bookings Over Time: Area chart showing the last 30 days of booking volume.
  • Revenue Trend: Area chart illustrating daily revenue for the past 30 days.
  • Top Events by Bookings: Bar chart of the five events with the highest booking counts.
  • Recent Activities: Feed of the latest ten booking, refund, and registration actions.
Refreshing Dashboard Data: The dashboard data loads on page entry and can be refreshed manually. To manually refresh, click the refresh icon at the top right.

Customer Management

Use the Customer Management screen to review registered customers, update their account status, and export filtered lists for reporting.

What You Can See

Each row in the customer table displays:

  • Customer name and email verification badge (✅ Verified / ⚠️ Unverified)
  • Email address and phone number
  • Account status (Active, Blocked, Pending Verification)
  • Date the customer joined the platform
Access Control: Only users with Admin or Super Admin roles (or the customers.view/customers.manage permissions) can access this screen.

Searching & Filtering

  • Search: Use the search box to find customers by name or email.
  • Status Filter: Filter the table to show All, Active, Blocked, or Pending Verification accounts.
  • Pagination: Results are shown 20 per page—use the Previous/Next buttons to browse.

Status Badges

Status Meaning
Active Customer can sign in and make bookings.
Blocked Access revoked—customer cannot log in or place new bookings.
Pending Verification Awaiting email verification or admin approval.

Blocking a Customer

  1. Click the Block icon on an Active customer.
  2. Provide a detailed reason in the confirmation modal (required, max 1000 characters).
  3. Confirm to immediately revoke access.
Auditing: Blocking actions are logged with the admin user ID and reason for compliance tracking.

Re-activating a Customer

  1. Locate a Blocked customer in the table.
  2. Click the Activate icon to restore access.
  3. The account status updates immediately; no additional confirmation is required.

Exporting Customers

  1. Apply any search terms or status filters.
  2. Click Export to download a CSV snapshot of the filtered list.
  3. The file includes IDs, contact information, status, and booking spend totals.
The export endpoint also supports JSON output (?format=json) and strips sensitive fields such as password hashes before delivery.

Cancellation Requests

Monitor and act on customer cancellation requests. The header includes a manual Refresh button to pull the latest queue from the API.

Queue Overview

  • Stats Cards: Summaries for Total, Pending Review, Approved, and Declined requests update after each action.
  • Status Badges: Table rows display Pending ⏳, Approved ✅, or Declined ❌ indicators for quick scanning.
  • Pagination: Requests are listed 20 per page with Previous/Next navigation.

Filtering & Search

  • Filter by status (All, Pending, Approved, Declined) to focus on the queue that matters.
  • Use the search box to match booking references, event names, or customer details.
  • Filters are sent to /admin/cancellation-requests with pagination parameters, so the results and counters stay in sync.

Reviewing a Request

  1. Click the View icon in the table to open the detailed modal.
  2. Review booking details (event, amounts, payment status), customer contact information, and the customer’s cancellation reason.
  3. If the request was previously handled, you will also see the reviewer, timestamps, and prior admin notes.

Approving a Request

  1. In the modal, choose Approve Cancellation.
  2. Add optional admin notes to explain the decision (stored with the request record).
  3. Click Confirm Approval. The system records the reviewer, applies the configured refund policy, and updates the booking’s cancellation status.
Approval triggers PATCH /admin/cancellation-requests/{id}/approve, which calculates default refund amounts and marks the request as approved while leaving refund processing in a "pending" state until settled.

Declining a Request

  1. Select Decline Request from the modal.
  2. Provide a reason of at least 10 characters—this is required by the API for auditing.
  3. Confirm the decline to notify the customer and reset the booking’s cancellation status.
Declines are validated server-side via PATCH /admin/cancellation-requests/{id}/reject; missing or short notes are rejected before anything changes.
For exceptional cases (medical, force majeure, organizer cancellations), capture detailed admin notes so the activity log clearly explains the resolution.

Refund Management

Refund Queue Overview

  • Monitor all customer refund requests in a paginated table that shows refund reference, booking reference, customer details, reason, amount, status, and request date.
  • Use the search box to instantly filter by refund reference, booking reference, customer name, or refund reason.
  • Status chips and action buttons let you understand queue health at a glance and respond without leaving the table.

Refund Statuses

Pending Under Review Approved Processed Completed Rejected Failed

Status flow: Pending → Under Review → Approved → Processed/Completed or Failed. Rejected requests end the workflow. Failed refunds can be re-tried from the API and will move back into Approved/Processed once successful.

Searching & Filtering

  • Status Filter: Narrow the queue to Pending, Under Review, Approved, Rejected, Processed, Completed, or Failed requests.
  • Date Range: Apply start/end dates to focus on refunds requested within a specific period.
  • Keyword Search: Match refund reference, booking reference, event name, or customer name in real time.
  • Filters are sent to the /admin/refunds endpoint with pagination parameters so totals stay accurate.

Reviewing Refund Requests

  1. Click View on any refund row to open the detailed modal.
  2. Review booking information, customer contact details, refund history, and any existing admin notes.
  3. Add internal admin notes to document context or follow-up actions before making a decision.

Updating Refund Statuses

  • Under Review: Move pending refunds into triage to signal another admin is investigating.
  • Approve: Approvals can optionally include an approved amount. Approved values must be numeric, non-negative, and cannot exceed the customer’s requested amount.
  • Reject: A rejection reason (up to 1000 characters) is mandatory. Customers will see this explanation in their notification.
  • Processed / Failed: Stripe webhooks or manual follow-up updates move approved refunds into their final state. Failed attempts can be retried after correcting the root cause.
  • All status changes are captured in the activity log with admin ID, timestamps, and metadata for auditing.
Field limits: Admin notes support up to 2000 characters. Approved amounts over 999,999.99 or negative values are rejected by the API.

Quick Actions

From the table view, Pending refunds expose inline Approve and Reject buttons for rapid triage. These apply standard validation rules and refresh the queue once complete.

Banners & Promotions

Overview

The Banners & Promotions module allows you to manage promotional banners displayed throughout the customer-facing website. Create eye-catching promotions for events, special offers, and announcements with full control over scheduling, placement, and performance tracking.

Banner Locations

Homepage Hero

Large hero banner displayed prominently on the homepage. Ideal for major promotions and featured events.

Homepage Secondary

Smaller banners below the hero section. Perfect for secondary promotions and additional announcements.

Category Page

Banners displayed on specific category pages. Target promotions to relevant event categories.

Event Page

Banners shown on individual event pages. Promote related events or upsell opportunities.

Creating a New Banner

  1. Navigate to Content Management → Banners & Promotions
  2. Click the "➕ Create Banner" button
  3. Basic Information: Enter a descriptive title and optional description
  4. Location: Choose where the banner will be displayed
  5. Image Upload: Upload banner image (JPG, PNG, or GIF - max 2MB)
  6. Link Settings: Add destination URL and choose link target (same/new window)
  7. Scheduling: Set start date and optional end date for automatic display
  8. Display Order: Set position number (lower numbers appear first)
  9. Status: Set to Active, Inactive, or Scheduled
  10. Price Tag (Optional): Add promotional pricing text
  11. Click "Save" to create the banner
Pro Tip: Use descriptive titles for easy identification in the admin panel. Titles are not visible to customers.

Managing Existing Banners

View Options:

  • Search: Find banners by title or description
  • Filter by Status: Active, Inactive, or Scheduled
  • Filter by Location: Show banners for specific display areas
  • Pagination: Navigate through large banner collections

Quick Actions:

  • Edit: Modify banner details, image, or scheduling
  • Delete: Remove banners permanently (requires confirmation)
  • Toggle Status: Quickly activate/deactivate banners

Drag & Drop Sorting

Easily reorder your banners using the intuitive drag and drop interface:

  1. Hover over the handle () on the left side of any banner in the list
  2. Click and hold the handle, then drag the banner to its new position
  3. A visual guide will show the new position as you drag
  4. Release to drop the banner in its new position
  5. The display order will update automatically
Tip: The banner at the top of the list will appear first in the frontend. Use the preview function to see your changes in real-time.

Sorting Behavior:

  • Banners are sorted by their display order (ascending)
  • Banners with the same display order are sorted by creation date (newest first)
  • Changes are saved automatically when you drop a banner

Banner Scheduling

Status Types:

  • Active: Currently displayed to customers (within schedule dates)
  • Inactive: Not displayed, regardless of schedule
  • Scheduled: Will automatically display based on start/end dates

Automatic Scheduling:

  • Set Start Date to automatically activate banners at specific times
  • Set End Date to automatically deactivate banners
  • Banners outside their scheduled dates remain hidden even if set to Active
Important: Always review end dates for time-sensitive promotions to avoid displaying expired offers.

Image Management

Content Strategy:

Common Issues & Solutions

Banner Not Displaying:

  • Check if status is set to "Active"
  • Verify start/end dates are correct
  • Ensure position number doesn't conflict with other banners
  • Confirm location is appropriate for the page being viewed

Image Not Loading:

  • Verify image file size is under 2MB
  • Check image format (JPG, PNG, GIF only)
  • Try re-uploading the image
  • Clear browser cache and refresh

Low Click-Through Rate:

  • Review banner design and call-to-action clarity
  • Check if link destination is relevant and working
  • Consider banner positioning and visibility
  • A/B test different designs and messaging
Success Strategy: Effective banner management combines compelling visuals, strategic placement, precise scheduling, and continuous performance optimization. Regular analytics review helps identify what resonates with your audience.
